## Featured projects

### [ESP32_Thermostat](https://github.com/AlexMihai1804/ESP32_Thermostat)
A smart thermostat platform for room-based monitoring and control built on ESP32, BLE sensors, and Wi-Fi connectivity.
It includes a local web interface for configuration, live monitoring, and device control without relying on cloud services.
The project combines embedded firmware with a practical interface, making it closer to a complete system than a standalone prototype.

**Stack:** `C/C++` `ESP32` `BLE` `Wi-Fi` `HTML/CSS/JavaScript`

### [Yeelight](https://github.com/AlexMihai1804/Yeelight)
An ESP32 / Arduino library for local Yeelight LAN control, designed for fast communication and low-latency response.
The goal was to keep device control lightweight, local, and easy to integrate into automation setups.
It reflects my interest in building tools that are simple, useful, and close to the hardware.

**Stack:** `C/C++` `ESP32` `Arduino` `LAN/UDP`

### [PixelSync](https://github.com/AlexMihai1804/PixelSync)
A desktop application that captures on-screen content and syncs it with RGB lighting for ambient effects.
It was built with multi-monitor setups, smooth color updates, and responsive performance in mind.
The project highlights the software side of my work, especially applications that connect user experience with hardware behavior.

**Stack:** `Python`

### [AutoWatering](https://github.com/AlexMihai1804/AutoWatering)
A modular irrigation controller built around nRF52840 and Zephyr RTOS for multi-zone water control.
It combines BLE configuration, scheduling, and sensor-driven logic in a system meant for real deployment rather than a simple prototype.
I am currently developing it further to reduce energy consumption and make irrigation adapt better to temperature and precipitation.

**Stack:** `C/C++` `Zephyr RTOS` `nRF52840` `BLE`

### [digital-filter-bank-sv](https://github.com/AlexMihai1804/digital-filter-bank-sv)
A configurable SystemVerilog digital filter bank for conditioning asynchronous inputs through per-channel synchronization, programmable filtering, and interrupt generation.
It processes 17 independent channels through a shared register-controlled architecture, making it useful for debounce-style qualification, glitch rejection, and cleaner digital signal handling.
The project reflects my interest in digital design and verification, especially building practical RTL blocks with clear behavior and dedicated testbenches.

**Stack:** `SystemVerilog` `RTL Design` `Digital Design` `Verification`
